Hello, Bonjour, Merhaba we are FireWatch, we provide bushfire location information in a quickly accessible form..

    About FireWatch

    FireWatch is an application that allows an admin of a forest park, or nature reserve to give their rangers the ability to be more aware of what's going on around them.


    Currently we have allocated for there to be four sectors of a major park to be observed. When a temperature spike is picked up by one of our Rasberry Pi's the ranger over that sector will be alerted through the application and able to identify the issue quickly and report back to the application that it has been resovled.


    The FireWatch website is used to gather this data to keep note of the occurances of fires throughout the park, what rangers belong in what section and just to give overall information about FireWatch.

    About the Team

    FireWatch was created as our third year group project, where we had to create a project of our own choice so long as it was three tiered.

    At the time of being asked this there was a horrendous bushfire sweeping through Australia.


    The fire situation in Australia worsened significantly at the beginning of Nov. 2019. In mid-January 2020, a wave of heavy rain finally brought relief in some areas hit by bushfires but was nowhere near sufficient to extinguish the fires. It did provide a respite for firefighters in some areas who had seen only occasional breaks in the weather in the past few months. Hotter temperatures, drought and high winds escalated the crisis again over the first weekend in February. In mid-February, an extremely heavy rain storm allowed firefighters to contain all of the fires in New South Wales (NSW) although fires continued to burn in Victoria.


    As we watched this occur, we felt certain that we would base or project around what was happening, so this is where the idea of Firewatch came from.


    Our team consists of Samantha O'Sullivan, Kevin Mulraney, Daniel McGlynn and Kevin Troy. We are third year students in Athlone Institute of Technology and are currently studying Software Design in Mobile Applications and Connected Devices.

    What Technologies were used to make this possible?

    MQTT

    What is it?

    MQTT is a lightwight machine-to-machine (M2M)/"Internet of Things" connectivity protocol. We used MQTT to connect our sensor to our android application.


    Why use MQTT?

    We used MQTT for its ease of implimentation, reliability and responsiveness. MQTT is lightweight, simple to understand and very powerful given its footprint Projects that use sensor data often use mqtt when they want to reduce their code footprint.


    Android Studio

    What is it?

    Android Studio is the official integrated development environment for Google's Android operating system It allows for easy debugging and testing while being a powerful tool to create exactly thte app you need no matter your project. For FireWatch, android studio was perfect. We leveraged our experience developing for android studio to bring the best app we could


    Why use Android Studio?

    Android studio provides tools to help with the layout of your app while being a powerful tool for developers that is easy to debug. Android studio supports MQTT and this was perfect for our needs


    Python & Raspberry pi SenseHat

    What is it?

    Python is a high-level, general-purpose programming language that we used for coding the SenseHat on the Raspberry pi to simulate forest fire data. The Raspberry pi is a single small board computer, useful for small projects such as home automation in the IOT.


    Why use Python?

    Python can be used to code the SenseHat LED board attachment for the Pi. We used Python to simulate live temperature sensor data. Python is a powerful, general purpose programming language.
